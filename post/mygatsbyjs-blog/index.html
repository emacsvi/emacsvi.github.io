Gatsby博客

花了一天一夜的时间把博客从以前的jekyll迁到了gatsbyJs的环境了。原因有如下几个： - 张恋哥哥说之前的网页比我本人还丑。 - Gatsby的单页面打包成bundle之后的访问速度真的快了很多。 - 支持分页 - 支持标签 - 支持分类 - 支持图片懒加载 - 支持草稿功能 - 支持react很多插件 - …

一些报错

其实在迁移的过程中大部分时间在找主题上，想找一个比较合适自己的，说实话真的好难。没有发现完美的，自己又不会创作前端的页面。真的是悲哀。

看看frontmatter的示例:

---
title: "gatsbyJs博客"
slug: "url-path-write-here"
date: "2018-11-23 01:00:01"
categories:
    - go
tags:
    - 译文
    - 技术
    - nodejs
    - go
    - javascript
draft:true
---
  • 因为支持草稿，代码里面没有做优化，所以需要至少有一篇文章的frontmatter要写上：draft:true的标识。
  • 在md里面如果存在不支持的frontmatter也会报错的。
  • 遇到TypeError: getNodesByType is not a function错误的时候我找了很久，实际上是gatsby不兼容的问题。奶奶的nodejs版本问题真的恶心死人。解决办法是：yarn add gatsby
  • 代码的高亮是利用prismjs插件来做的，听说很小巧而且简单无比。css样式自行到prismjs上面去定制即可。但是感觉样式比起google的少之又少，而且还很丑。
  • 代码高亮在使用时语言的标签如何写，请看文末的链接prismjs supported languages。
  • 支持的常用语言标签有：go, python, markdown,docker,json,sql,bash,javascript,yaml,ini,html

感想

突然觉得我应该多花点时间在写作上，虽然不能像何大娃那样成为一名诗人，但是写点自己的生活点点滴滴表示自己还在努力着就好。

自动化

每次想要更新到外网的话只需要运行如下bash脚本即可：

gitDir=/Users/liwei/Documents/repo/simple
blogDir=/Users/liwei/coding/react/yang
cd $blogDir
echo "开始build静态网页..."
gatsby build
echo "复制到github仓库目录..."
cp -rf public/* $gitDir/
cd $gitDir
git add .
DATE=`date '+%Y-%m-%d %H:%M:%S'`
echo "git commit仓库..."
git commit -a -m "update by liwei ${DATE}"
echo "上传到github仓库..."
git push origin master
cd $blogDir
