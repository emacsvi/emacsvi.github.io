frp内网穿透

2019年4月1日
| go

家里有台式机24*7开机，用来当服务器。这样可以在外网访问，做一些集群化的工作。减轻笔记本上面呼呼喘息的声音(经常在mbp上跑十几个docker container,真的对它太不珍惜了，现在用内网的机器跑起来就踏实多了)。你可以在一个台式机上用virtualbox装几个linux虚拟机，当服务器用，比便宜的vps可强太多了。我的台式机i5 8500 + 16g内存 + 256gSSD的配置，装了3个虚拟机(每个机器4个cpu + 4g内存)当服务器，很强悍。而且现在我随处走到哪儿，即便是在咖啡厅装B也可以随时连接上我家里的服务器集群做想做的事情(不过我从来没有在咖啡厅去过)。现在有5台机器连接在集群下面的。打算等内存和cpu便宜了再升级一下台式机，多装上几个虚拟机，这样要不要太爽了。之前我跑的kubernetes就跑在这上面的。你也来试试吧！

概念

内网穿透大家应该都知道是什么东西，很久以前用的花生壳，DDNS的东西，后来又用了ngrok，ngrok也是go语言写的。但是2.x之后不开源了。1.x据说不太稳定。又出来一个frp的，也是go语言写的，个人感觉非常好用。也很稳定快速。这里记录一下。以免忘记。

我的环境如下（也可以说是准备工作）：

  • 一台腾讯云的VPS，有外网ip地址可以访问。frps服务器的程序就跑在这上面。
  • 家里有个台式机器做客户端。上面跑frpc程序与服务器侧的frps进行内网连接。

frps服务器端的配置

下载二进制文件： https://github.com/fatedier/frp/releases 将二进制文件放到VPS的外网服务器上。

准备好配置文件，内容如下：cat frps.ini

[common]
bind_port = 7000
token = xxx
authentication_timeout = 900
dashboard_port = 7500
dashboard_user = admin
dashboard_pwd = admin
vhost_http_port = 10080
vhost_https_port = 10443

然后修改开机启动文件的权限，并且加入开机启动中。

chmod +x /etc/rc.local
nohup /root/frp/bin/frps -c /root/frp/bin/frps.ini &

查看服务器的dashboard界面：http://bc:7500/static/

我将bc修改了host将vps的外网ip映射过来。并且连接了5台内网电脑在这上面，因为我在内网做了一个集群。一直跑着。下面是效果图：

frpc客户端的配置

客户端安装与服务器类似，但是一定要记住，不同的客户端连接到服务器时配置文件中[名称]一定不能一样。否则只能有一个连接成功。

客户端的配置cat frpc.ini：

[common]
server_addr = 118.126.116.28
server_port = 7000
token = xxx

[ssh110]
type = tcp
local_ip = 127.0.0.1
local_port = 22
use_encryption = false
use_compression = false
remote_port = 6110

同样将开机启动加入/etc/rc.local之中。并且将/etc/rc.local改为可执行权限才能生效。

chmod +x /etc/rc.local
# 并且写入/etc/rc.local
nohup /root/frp/frpc -c /root/frp/frpc.ini &

测试

最后测试连接是否成功(bc就是我的vps的外网ip)：

# 用小写p可以直接登录
ssh -p 6110 root@bc
# 或者用-oPort参数
ssh -oPort=6110 root@bc

# 复制文件用大P
scp -P6110 frp-dashboard.png root@bc:/root/

# 参数p后面可以有空格也可以没有

# 复制你的公钥上去免密码登录
ssh-copy-id -P 6118 liwei@bc

拷贝大文件时候出错：

write failed: Broken pipe
lost connection
  • 方法一：如果您有多台服务器，不想在每台服务器上设置，只需在客户端的 ~/.ssh/ 文件夹中添加 config 文件，并添加下面的配置：ServerAliveInterval 60
  • 方法二：如果您有多个人管理服务器，不想在每个客户端进行设置，只需在服务器的 /etc/ssh/sshd_config 中添加如下的配置： ClientAliveInterval 60
  • 方法三：如果您只想让当前的 ssh 保持连接，可以使用以下的命令： ssh -o ServerAliveInterval=60 user@sshserver

开机启动

刚刚说了centos7的开机启动方式，需要将/etc/rc.local改了可执行权限。然后将启动命令写入这个文件里面即可。

而ubuntu18.04的启动已经不一样了。ubuntu18启动脚本 后面还需要启动

# 1. 建立启动软连接
ln -s /lib/systemd/system/rc.local.service /etc/systemd/system/ 

# 2. 修改/lib/systemd/system/rc.local.service内容，增加如下一段安装的话：
[Install]  
WantedBy=multi-user.target  
Alias=rc-local.service

# 3. 新创建/etc/rc.local文件 并且给755权限
# sudo vim /etc/rc.local 内容如下：
#!/bin/sh -e
#
# rc.local
#
# This script is executed at the end of each multiuser runlevel.
# Make sure that the script will "exit 0" on success or any other
# value on error.
#
# In order to enable or disable this script just change the execution
# bits.
#
# By default this script does nothing.
nohup /home/liwei/frp/frpc -c /home/liwei/frp/frpc.ini &
exit 0

# 4. 给权限
sudo chmod 755 /etc/rc.local

# 5. 启动服务
sudo systemctl enable rc-local

# 6. 启动服务并检查状态
sudo systemctl start rc-local.service
sudo systemctl status rc-local.service

# 7. 最后重启试试
sudo init 6

ssh -p 6118 liwei@bc
